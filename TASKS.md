## Известные баги

- [x] race condition в push-уведомлениях (getInitialMessage и onMessageOpenedApp разведены между SplashScreen и main.dart, см. комментарии в обоих файлах)
- [x] незащищённые вызовы FCM (district_provider.dart: _subscribeSafely/_unsubscribeSafely)
- [x] фейковая кнопка "закладка" (теперь персистится через LocalStorageService)
- [x] отсутствовали composite-индексы для announcements(district+promotedUntil) и events(district+eventDate) — промо-объявления и афиша могли не грузиться в проде; добавлены в firestore.indexes.json, требуют `firebase deploy --only firestore:indexes`
- [x] падение приложения при открытии галереи (объявления/организации): Hero-теги строились из URL картинки; если в галерее случайно встречались два одинаковых URL (ручной ввод в админке), Flutter падал с "multiple heroes that share the same tag". Теги теперь строятся из URL+индекс через FullscreenGalleryViewer.heroTag — исправлено во всех 4 экранах деталей (announcement/organization/event/news) и самом просмотрщике
- [x] кнопка "Объявления" в _QuickNavRow на главном экране ломала выравнивание ряда — самая длинная подпись из четырёх переносилась на вторую строку без ограничения maxLines, из-за чего кнопка становилась выше соседних. Подпись теперь в одну строку через FittedBox
- [x] раздел "Рекламный баннер" в админке (docs/index.html) падал с ошибкой при загрузке: на проде в Firestore был развёрнут устаревший индекс sponsored_content(district+createdAt), тогда как код уже давно фильтрует/сортирует по activeUntil (с партии 3.4/3.6). Индекс не совпадал с запросом → "requires an index". Задеплоен актуальный индекс из firestore.indexes.json (`firebase deploy --only firestore:indexes`); заодно добавлен в файл недостающий индекс ad_requests(district+status+createdAt) — он существовал на проде, но не был отслежен в репозитории
- [x] админка (docs/index.html) на мобильных: поля ввода имели font-size 15px — iOS Safari зумит страницу при фокусе на любом поле с font-size < 16px, из-за чего вёрстка выглядела "съехавшей" после каждого тапа в форму. Поднято до 16px. Также: `.top-bar` (заголовок + "Выйти") — несовпадающие margin у h1 и button внутри align-items:center визуально сдвигали их друг относительно друга; `.item`-строки списков не имели защиты от переполнения (flex-wrap, min-width:0, word-break) — длинный заголовок/ссылка могли вытолкнуть кнопки "Изменить"/"Удалить" за край экрана
