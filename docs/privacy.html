ОМСКРЕГИОН ИНФО

Политика конфиденциальности

Последнее обновление: 24 июля 2026 г.

1. Общие положения

Настоящая Политика конфиденциальности (далее — «Политика») описывает, какие данные обрабатывает мобильное приложение «ОМСКРЕГИОН ИНФО» (далее — «Приложение»), с какой целью и как эти данные защищаются.

Приложение разработано и администрируется физическим лицом (далее — «Оператор»). Политика составлена в соответствии с требованиями Федерального закона от 27.07.2006 №152-ФЗ «О персональных данных».

Приложение работает без регистрации и входа в аккаунт — у большинства функций нет привязки к личности пользователя.

2. Какие данные обрабатываются

2.1. Номер телефона

Номер телефона запрашивается добровольно, только когда житель сам решает воспользоваться одной из следующих функций:

  • подача объявления в разделе «Объявления»;
  • подача заявки на размещение вакансии;
  • подача заявки на размещение рекламного баннера;
  • обращение к администрации приложения (это поле необязательно к заполнению).

Номер используется исключительно для связи с заявителем и модерации заявки.

Важно: если бесплатное объявление или вакансия проходит модерацию и публикуется, указанный контактный телефон становится виден всем пользователям Приложения — это необходимо, чтобы заинтересованные жители могли связаться с автором объявления напрямую. Номер телефона, указанный при подаче заявки на баннер или обращении в поддержку, третьим лицам не показывается — виден только администратору для связи по вопросу заявки.

2.2. Локальный идентификатор устройства

Приложение генерирует случайный технический идентификатор (набор случайных символов), который сохраняется на устройстве. Этот идентификатор:

  • не позволяет установить личность пользователя — это не номер телефона, не IMEI и не рекламный идентификатор устройства, а случайно сгенерированный набор символов;
  • используется для того, чтобы ограничить одну оценку организации с одного устройства, показать пользователю историю его же заявок («Мои заявки») и защититься от повторной автоматической отправки одного и того же обращения;
  • удаляется при удалении приложения или сбросе его данных — новый идентификатор при этом создаётся заново.

2.3. Push-уведомления

Приложение использует сервис доставки push-уведомлений (Firebase Cloud Messaging) для уведомления жителей о новостях района, публикации объявлений и ответах на обращения в поддержку. Технический идентификатор устройства для push-рассылки обрабатывается инфраструктурой сервиса уведомлений автоматически и не передаётся и не сохраняется Оператором отдельно.

Часть push-уведомлений жители могут включать и отключать самостоятельно в настройках Приложения.

2.4. Геолокация

Приложение не запрашивает и не собирает данные о геолокации устройства. Координаты, которые используются для отображения погоды, определяются по названию района, который житель выбирает вручную при первом запуске — не по фактическому местоположению устройства.

2.5. Фотографии

Если житель прикладывает фотографии к объявлению или заявке на баннер, эти фотографии загружаются в облачное хранилище для отображения в Приложении. Загружаются только те фотографии, которые пользователь выбрал сам и добровольно приложил к заявке.

3. Передача данных третьим лицам

Оператор не передаёт и не продаёт данные пользователей третьим лицам в маркетинговых или иных коммерческих целях.

Для работы Приложения используются технические сервисы, обеспечивающие хранение данных, доставку push-уведомлений и хранение изображений. Эти сервисы обрабатывают данные только в объёме, необходимом для оказания соответствующей технической услуги, и не используют их в собственных целях.

4. Хранение данных

Данные хранятся в течение времени, необходимого для целей, указанных в разделе 2 (например, объявления автоматически скрываются из ленты Приложения по истечении установленного срока). Часть данных (закладки, выбранный район, история заявок пользователя) хранится только локально на устройстве пользователя и не передаётся Оператору.

5. Права пользователя

Поскольку Приложение работает без регистрации, большинство данных, относящихся к конкретному пользователю, можно удалить самостоятельно — удалением Приложения или очисткой его данных в настройках устройства.

Для запроса на удаление или уточнение данных, хранящихся на стороне Оператора (например, поданной заявки на объявление), обращайтесь по контакту, указанному в разделе 6.
